
## 一、创建和删除虚拟内存

### ① 创建

1. **创建swap文件**：`sudo dd if=/dev/zero of=/swapfile bs=1M count=2048` 使用 dd 命令分配 2GB 的空间（这里 count=2048 代表 2GB，因为块大小 bs=1M）
2. **设置权限**：`sudo chmod 600 /swapfile` 为了安全，Swap 文件必须只能由 root 用户读写
3. **将其格式化为 Swap 格式**：`sudo mkswap /swapfile`
4. **启用 Swap**：`sudo swapon /swapfile`
5. **验证是否生效**：输入`free -h`，应该能看到 Swap 行的数值增加了
6. **永久生效（防止重启后失效）**：`echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab`

### ② 删除

1. **关闭 Swap**：`sudo swapoff /swapfile`
2. **移除 fstab 中的配置**：`sudo nano /etc/fstab` 找到包含 /swapfile 的那一行，将其**删除**（或者在行首加 # 注释掉）。保存并退出
3. **删除物理文件**：`sudo rm /swapfile`

### ③ 修改大小

只能先删除再创建。执行 swapoff 前，确保系统物理内存（RAM）足够，否则关闭的一瞬间如果内存溢出，系统可能会直接崩溃或导致运行中的服务被杀死


## 二、虚拟内存策略
### ① 原理
**vm.swappiness**的取值为0-100。数值越小，系统越倾向于使用物理内存；数值越大，系统越倾向于把暂时不用的内存数据扔进 Swap

- **0**：除非物理内存彻底耗尽，否则绝不使用 Swap
- **10**：对于小内存机器，gemini个人推荐设置为 10
- **60**：默认值。较为均衡
- **100**：内核会非常频繁、非常积极地把物理内存中暂时不用的数据（匿名内存，即程序运行时的堆栈空间）移动到 Swap 分区中
	- **内核逻辑：** 内核会认为：“只要这个内存页在过去几秒内没有被读写过，那就把它扔到 Swap 里去，哪怕我现在的物理内存还有大把空余”
	- **目的是：** 为了尽可能多地保持物理内存的**完全空闲（Free）**，或者为文件系统的缓存（Cache）腾出更多空间
	- **设置为 100 的后果（为什么不建议这样做）**：虽然物理内存确实空出了很多，但由于频繁地将数据从内存搬运到磁盘（Swap），系统会产生大量的 **I/O 等待**。你的服务器会变得极其卡顿，甚至在执行简单的 ls 或 cd 命令时都要卡一下，因为系统正在忙于“倒腾”内存，长期这样运作会显著缩短 SSD 的寿命

### ② 管理

- **查看当前生效的策略**：`cat /proc/sys/vm/swappiness`
- **修改策略立即生效**：`sudo sysctl vm.swappiness=10`
- **修改策略永久生效**：`sudo vi /etc/sysctl.conf`编辑文件，新增或修改`vm.swappiness=10`，保存退出后，执行`sudo sysctl -p`使其生效
